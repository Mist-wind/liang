# 部署说明

## 问题解决

如果运行PowerShell脚本时遇到以下错误：

```
无法加载文件 C:\Users\ikingtec\Desktop\03个人博客\deploy.ps1，因为在此系统上禁止运行脚本。
```

这是因为Windows的PowerShell执行策略限制。有以下解决方案：

### 方案1：使用批处理脚本（推荐）

直接双击运行 `deploy.bat` 文件，批处理脚本通常不会受到执行策略的限制。

### 方案2：临时更改PowerShell执行策略

1. **以管理员身份运行PowerShell**
   - 搜索"PowerShell"，右键点击"Windows PowerShell"，选择"以管理员身份运行"

2. **执行以下命令**：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

3. **确认更改**：
   - 输入"Y"并按Enter键

4. **运行部署脚本**：
   ```powershell
   .\deploy.ps1
   ```

5. **部署完成后，建议恢复执行策略**：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy Restricted -Scope CurrentUser
   ```

## 部署步骤

### 1. 安装Git（如果尚未安装）

- 从 https://git-scm.com/downloads 下载并安装Git
- 安装时确保选择"Add Git to PATH"选项

### 2. 初始化Git仓库（如果尚未初始化）

在项目目录中执行以下命令：

```bash
git init
git remote add origin https://github.com/Mist-wind/liang.git
git add .
git commit -m "Initial commit"
git push -u origin main
```

### 3. 设置GitHub Pages

- 进入GitHub仓库 → Settings → Pages
- 选择 `main` 分支作为Source
- 点击 "Save" 按钮

### 4. 运行部署脚本

- **Windows用户**：双击运行 `deploy.bat` 文件
- **PowerShell用户**：按照方案2的步骤运行 `deploy.ps1` 脚本

### 5. 访问网站

部署完成后，访问：https://mist-wind.github.io/liang/

## 常见问题

### Q: 脚本提示"Git未安装或未添加到系统PATH中"

**A:** 请从 https://git-scm.com/downloads 下载并安装Git，确保安装时选择"Add Git to PATH"选项。

### Q: 脚本提示"当前目录不是git仓库"

**A:** 请按照步骤2初始化Git仓库并添加远程仓库。

### Q: 推送失败，提示需要输入GitHub凭证

**A:** 请确保您已登录GitHub，并且有权限推送代码到仓库。可能需要配置Git凭证或使用SSH密钥。

### Q: GitHub Pages部署后访问404

**A:** 请等待几分钟，GitHub Pages需要时间部署。如果仍然无法访问，请检查GitHub Pages设置是否正确。

## 手动部署步骤

如果脚本仍然无法运行，您可以手动执行以下命令来部署：

```bash
# 添加所有更改
git add .

# 提交更改
git commit -m "Deploy update $(date)"

# 推送到GitHub
git push origin main
```
